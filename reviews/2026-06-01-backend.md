## P0 â€” Critical (must fix before merge)

### 1. Path traversal in production static-file server (`server.mjs:33`)

**File:** `/Users/nickaisbitt/AutoTube/server.mjs` line 33

```js
let filePath = join(DIST_DIR, req.url === "/" ? "index.html" : req.url);
```

`path.join` normalises `..` segments, so a request like `GET /../../etc/passwd` resolves to a path **outside** `DIST_DIR`. The subsequent `existsSync` / `readFileSync` will happily read and serve any file the process has access to. There is no `resolvedPath.startsWith(DIST_DIR)` guard.

**Impact:** Arbitrary file read on the production server. An attacker can exfiltrate `.env.local`, source code, `/etc/passwd`, etc.

**Fix:** After `join`, resolve the path and verify it starts with `DIST_DIR + "/"`:
```js
const resolved = resolve(filePath);
if (!resolved.startsWith(resolve(DIST_DIR) + "/")) { /* 403 */ }
```

### 3. CORS wildcard `*` set by every route handler overrides production CORS middleware

**Files:** All 20+ route handlers (e.g. `searchImages.ts:50`, `saveProject.ts:36`, `renderOutput.ts:59`, `proxyImage.ts:116`, etc.)

The CORS middleware in `server/middleware/cors.ts` correctly restricts `Access-Control-Allow-Origin` to an allow-list in production. However, **every individual route handler** subsequently calls `res.setHeader("Access-Control-Allow-Origin", "*")`, which **overwrites** the middleware's careful origin restriction.

**Impact:** In production, any website can make credentialed API requests to AutoTube. Combined with the SSRF-adjacent endpoints (`/api/proxy-image`, `/api/download-clip`), this enables cross-site data exfiltration.

**Fix:** Remove the per-route `Access-Control-Allow-Origin: *` calls. Let the CORS middleware be the single source of truth.

### 4. Rate limiter trivially bypassed via `X-Forwarded-For` spoofing

**File:** `/Users/nickaisbitt/AutoTube/server/middleware/rateLimiter.ts` line 27

```ts
(req.headers["x-forwarded-for"] as string)?.split(",")[0]?.trim()
```

The rate limiter trusts the `X-Forwarded-For` header unconditionally. If the server is directly exposed (no reverse proxy), an attacker can rotate this header per request to get a fresh rate-limit bucket every time.

**Fix:** Only trust `X-Forwarded-For` when behind a known reverse proxy (check `process.env.TRUST_PROXY` or similar). Otherwise, use `req.socket.remoteAddress` exclusively.

### 5. `renderOutput.ts` â€” Path containment check has off-by-one prefix bug

**File:** `/Users/nickaisbitt/AutoTube/server/routes/renderOutput.ts` line 48

```ts
if (!resolvedPath.startsWith(RESOLVED_PROJECT_ROOT) || !existsSync(resolvedPath)) {
```

`startsWith` is a string prefix check. If `RESOLVED_PROJECT_ROOT` is `/app/AutoTube`, a resolved path of `/app/AutoTubeXYZ/secret` would pass. The `..` and null-byte checks on line 26 mitigate the most obvious attacks, but the containment check is still technically incorrect.

**Fix:** Use `startsWith(RESOLVED_PROJECT_ROOT + "/")` or `resolvedPath === RESOLVED_PROJECT_ROOT`.

### 6. `qualityCheck.ts` â€” API key leaked via process argument list

**File:** `/Users/nickaisbitt/AutoTube/server/routes/qualityCheck.ts` lines 127-129

```ts
args.push("--api-key", apiKey);
```

The OpenRouter API key is passed as a command-line argument to the spawned Python process. On Linux, `/proc/<pid>/cmdline` exposes this to any user on the system. On macOS, `ps aux` shows it.

**Fix:** Pass the key via an environment variable instead:
```ts
env: { ...process.env, OPENROUTER_API_KEY: apiKey }
```

### 7. No OPTIONS preflight handler â€” cross-origin POST requests may fail

**File:** `/Users/nickaisbitt/AutoTube/server/index.ts`

The CORS middleware sets `Access-Control-Allow-Methods` and `Access-Control-Allow-Headers`, but there is no `OPTIONS` request handler. When a browser sends a CORS preflight `OPTIONS` request (required for cross-origin POST with `Content-Type: application/json`), it falls through to `next()` â†’ `serveStatic`, which returns `index.html` with a 200 status. The browser sees no CORS headers on the OPTIONS response and blocks the actual POST.

**Fix:** Add early return for `req.method === "OPTIONS"` after the CORS middleware runs.

### 8. `serverRender.ts` â€” No timeout on child render process

**File:** `/Users/nickaisbitt/AutoTube/server/routes/serverRender.ts` lines 107-175

The spawned child process has no timeout. If the render hangs (e.g., ffmpeg deadlock, network timeout in edge-tts), the child runs indefinitely, consuming CPU and memory. The heartbeat interval keeps the SSE connection alive forever.

**Fix:** Add a `setTimeout` that kills the child after a reasonable maximum (e.g., 10 minutes).

### 9. `imageSearch.ts:5-13` â€” `puppeteer-extra` type import may fail at compile time

**File:** `/Users/nickaisbitt/AutoTube/server/utils/imageSearch.ts` lines 5-13

The type annotation `typeof import("puppeteer-extra")` at line 5 requires `puppeteer-extra` to ship TypeScript declarations. The package is installed (confirmed in `package.json` and `node_modules`), but its type declarations may not resolve correctly under all `tsconfig` module settings (e.g., `moduleResolution: "bundler"`). At runtime, the dynamic `import()` works fine because the package exists.

**Diagnosis:** This is a TypeScript compilation friction issue, not a runtime breakage. The `getPuppeteer()` function is only called in the Google Images headless fallback path. If `tsc` emits an error here, it may prevent the build from completing.

**Fix:** Use a looser type: `let _puppeteer: any = null;` or add a `// @ts-ignore` with a tracking issue.

### 10. `security.test.ts` â€” TypeScript errors with `dns.lookup` mock

**File:** `/Users/nickaisbitt/AutoTube/server/__tests__/security.test.ts` lines 43-46, 73-76

```ts
const spy = vi.spyOn(dns, "lookup").mockImplementation((hostname, options, callback) => {
  const cb = typeof options === "function" ? options : callback as any;
  cb(null, [{ address: "8.8.8.8", family: 4 }] as any);
});
```

`dns.lookup` has **three overloaded signatures** in `@types/node`. `vi.spyOn` + `mockImplementation` cannot reconcile all overloads, producing a TS error. The test logic is correct at runtime; this is purely a type-system friction issue.

**Diagnosis:** Not a security issue. The underlying `validateURL()` and `isPrivateIP()` functions are well-implemented with proper DNS resolution, private-IP blocking, and metadata-service protection.

**Fix:** Cast through `any`: `vi.spyOn(dns, "lookup") as any).mockImplementation(...)`.

### 11. `renderProgress.ts` â€” POST handler accepts arbitrary JSON into global state

**File:** `/Users/nickaisbitt/AutoTube/server/routes/renderProgress.ts` lines 56-72

The POST handler parses the body and calls `updateRenderProgress(progressData)`, which does `{ ...renderProgress, ...progressData }`. Any authenticated or unauthenticated client can inject arbitrary keys into the global render-progress object. There is no validation of the expected shape.

**Impact:** Low-severity data pollution. An attacker could set `status: "rendering"` permanently, confusing the UI.

**Fix:** Validate the incoming JSON against an expected schema (whitelist allowed keys).

## P2 â€” Medium/Low (nice to have)

### 13. `renderVideo.ts` â€” No frame-count or per-frame size validation

**File:** `/Users/nickaisbitt/AutoTube/server/routes/renderVideo.ts` lines 20-37

The endpoint accepts an array of base64-encoded PNG frames. While the global body-size limit is 10MB (set in `index.ts`), there is no limit on the number of frames. A request with thousands of tiny frames could cause disk exhaustion in the temp directory or an extremely long ffmpeg run.

**Fix:** Cap `frames.length` (e.g., 1800 for a 60s video at 30fps) and validate each frame's base64 length.

### 14. Migration 002 not idempotent â€” `ALTER TABLE ADD COLUMN` fails if column exists

**File:** `/Users/nickaisbitt/AutoTube/server/migrations/002-add-workspaces.sql`

```sql
ALTER TABLE projects ADD COLUMN workspace_id TEXT DEFAULT 'default';
```

SQLite does not support `IF NOT EXISTS` for `ADD COLUMN`. If the migration tracking table (`_migrations`) is lost or reset, re-running this migration will fail with "duplicate column name". The migration runner (`migrate.mjs`) correctly tracks applied migrations, so this is only an issue in recovery scenarios.

**Fix:** Wrap in a conditional check or use a try/catch in the migration runner for `ADD COLUMN` errors.

### 19. `docs.ts` â€” Potential XSS if `openapi.yaml` is tampered with

**File:** `/Users/nickaisbitt/AutoTube/server/routes/docs.ts` line 41

The parsed YAML is serialised to JSON and interpolated directly into a `<script>` block. ### 20. `server.mjs` â€” `serveStatic` reads entire file into memory synchronously

**File:** `/Users/nickaisbitt/AutoTube/server.mjs` lines 42-46

```js
const data = readFileSync(filePath);
```

For large static assets (video files, bundled JS), `readFileSync` blocks the event loop. The `MIME_TYPES` map includes `.mp4` and `.webm`, suggesting video files may be served this way.

**Fix:** Use `createReadStream(filePath).pipe(res)` for streaming, or use a proper static-file middleware.
