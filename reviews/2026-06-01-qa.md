## P0 – Critical Gaps (Immediate Action Required)

1. **Missing End‑to‑End Regression Tests for Recent Bugs**
   - Bug #1234 (YouTube thumbnail download failure) is fixed in code but has **no corresponding e2e test** in `e2e/`. Add a Playwright test covering the download flow with mocked network errors.
   - Bug #1198 (Audio sync drift after subtitle generation) lacks a test in `src/__tests__/`. Create a unit test mocking the TTS service response and verifying timestamp alignment.

2. **Coverage Gaps in Core Service Layer**
   - `src/services/videoProcessor.ts` and `src/services/tts.ts` show **< 70 % line coverage** (see coverage summary). Critical branches handling error retries are untested.
   - `server/__tests__/api.routes.test.ts` does not hit the *rate‑limit* middleware; add tests for 429 responses.

3. **Flaky Tests Identified**
   - `e2e/videoUpload.spec.ts` intermittently fails on CI due to race condition with the mock S3 bucket. The test uses a fixed timeout of `5000ms`; increase to `10s` and add explicit `await` on the upload promise.
   - `src/__tests__/utils/dateUtils.test.ts` fails randomly when the system timezone changes. Mock the timezone using `process.env.TZ='UTC'` in a `beforeAll` hook.

## P1 – Important Improvements (Next Sprint)

- **Test Speed Optimisation**
  - Total CI test time is ~7 min. `vitest` runs 40 % of tests in parallel already, but the **e2e suite** consumes ~3 min. Split the e2e specs into two separate projects (smoke vs full) using `playwright.config.ts` projects feature.
  - Enable `--cache` for Vitest and ensure `coverage` collection is limited to changed files in PRs.

- **CI Readiness Enhancements**
  - Add a pre‑test step to start a Dockerised PostgreSQL container only when `server/__tests__` are detected (use `CI=true` env flag).
  - Ensure `playwright` browsers are installed in CI cache to avoid nightly download delays.

- **Test Quality Checks**
  - Enforce at least **80 % statement coverage** for any file touched in a PR (use `nyc` thresholds).
  - Introduce a lint rule (`jest/no-focused-tests`) to prevent `test.only` or `it.only` from reaching CI.
